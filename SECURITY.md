# 安全策略

## 支持的版本

以下版本会收到安全更新。请始终使用受支持的版本。

| 版本       | 支持情况               |
|------------|------------------------|
| 1.x.x      | :white_check_mark: 当前稳定版，支持安全更新 |
| 0.9.x      | :x: 已停止支持，请升级 |
| < 0.9      | :x: 不支持             |

## 报告漏洞

我们非常重视项目安全。如果您发现安全漏洞，请按以下流程报告：

1. **不要公开提交 Issue** – 请将漏洞详情发送至专用邮箱：`security@yourproject.com`  
   （可选：也可以使用 [GitHub Private Vulnerability Reporting](https://docs.github.com/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)）

2. 您会在 **24 小时内** 收到我们确认收到报告的回复，之后每 3 天更新一次处理进度。

3. 如果漏洞被确认：
   - 我们会分配内部编号，并在修复后发布安全公告。
   - 可能将您加入致谢名单（需征得您同意）。

4. 如果漏洞被拒绝：
   - 我们会给出详细理由（如：已存在、风险低、不属于本项目的安全边界）。

5. **不设漏洞赏金计划**（或请根据实际情况说明：有/无，奖金金额，是否需要法律协议等）

感谢您帮助提升项目安全性！
